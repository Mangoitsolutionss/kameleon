h1. Kameleon [chameleon]

"!https://secure.travis-ci.org/cs3b/kameleon.png?branch=0_2_0!":http://travis-ci.org/cs3b/kameleon

Kameleon is a high abstraction dsl for better* writting acceptance, integration and functional tests using Capybara.

It's not dependend on any ruby framework (can be used with Ruby On Rails, Sinatra, Padrino, etc). So far binding are only done for one test framework: Rspec.

h2. Setup:

Kameleon depends on:

* rspec
* capybara
* selenium && racktest drivers
* devise ( kameleon uses paths suplied by devise. So if you're using some other authentication solution, you might want to overwrite it )
* thin (if you want run your specs faster - add gem 'thin' to Gemfile in your app)


h3. Gemfile

gem 'kameleon'
gem 'capybara'
gem 'selenium-webdriver'
gem 'headles' # to run firefox on virtual frame buffer

h3. System

xvfb and firefox should be installed

h3. Rspec Configure
In your spec file add (do not load capybara - kameleon will do it in a slightly different way, for more info look into lib/kameleon/ext/rspec/dsl.rb)

<pre>
require 'kameleon/ext/rspec/all'
</pre>
